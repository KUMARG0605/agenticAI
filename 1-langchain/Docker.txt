1. Container Management

			Life Cycle of the Container
			
				1. #docker run <containerName> <Docker-Image>
			1. Docker Doemon : --> checks and throws errors
							   --> responsible for creating all the stuff of the container
			once exicuted by default it have:
				1.Image checks in docker registery
				2.creates a Container
				3.unique id
				4.Name:
				5.Start application
				6.Script/Process
			
			Note: Until unless your process is running/alive in the container 
				  then the container is also alive
				
				
				
	1. how to remove a container ? 
	Note:  we cant delete the running container
		# docker rm <containerName / id >
		
	2.how to remove multiple containers ?
		#docker rm $(docker ps -aq)
		
	3. to run the container in backend / detached mode 
	   cmd:	#docker run -d --name sktech ubuntu
	   
	 # forcefully stoping the container
		docker stop <container - name >
		$
		
		docker ps --> only shows the running containers 
		docker ps -a --> shows all exited /up / created containers
